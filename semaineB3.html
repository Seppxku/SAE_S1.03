Où avons-nous, et sous quelle forme stocké la sauvegarde de la machine virtuelle créée?

Notre machine virtuelle ayant été créée avec VirtualBox est sauvegardée sous forme de fichiers (disques virtuels, fichiers de configuration) dans des répertoires dédiés à chaque utilisateur.

2. Création d'un compte administrateur.

Une fois la machine virtuelle prête à l’emploi, nous avons créé notre compte administrateur :

Voici les commandes décortiquées :

  • sudo permet d'exécuter la commande avec des privilèges administrateur, nécessaires pour gérer les utilisateurs.
  • useradd est la commande utilisée pour ajouter un utilisateur sur le système.
  • -m indique que le répertoire personnel de l'utilisateur doit être automatiquement créé. Par exemple, si l'utilisateur s'appelle alice, un dossier /home/alice sera créé.
  • -s permet de spécifier le shell par défaut que l'utilisateur utilisera après connexion.

La création du compte administrateur s’est faite grâce à la commande sudo SuperUser DO, qui est une commande qui nous sera utile tout le long de notre installation du poste de développement. Cette commande permet d'exécuter des actions qui nécessitent des autorisations administratives, comme l'installation de logiciels, la modification des fichiers système ou la gestion des utilisateurs.

Puis nous avons fait de cet utilisateur un administrateur en l’ajoutant au groupe sudo afin qu’il puisse avoir accès aux privilèges d’administration.

Voici les commandes décortiquées :

  • usermod permet de modifier un utilisateur existant (changer ses paramètres, ses groupes, son shell, etc.).
  • -a ajoute l'utilisateur aux groupes spécifiés sans retirer cet utilisateur des groupes auxquels il appartient déjà.
  • -G Spécifie les groupes secondaires auxquels l'utilisateur doit être ajouté.
